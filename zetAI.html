AI Credit Scoring - Credit Risk Assessment

Introduction

Zest AI is a fintech company that leverages artificial intelligence to revolutionize credit scoring. Their goals include:

  • Fraud Detection
  • High Accuracy
  • Automation and Efficiency
  • Fairness & Transparency

Data Collection & Encoding

One-Hot Encoding

Converts categorical variables into multiple columns, each represented as 0 or 1. Suitable for a small number of categories but creates many columns if the dataset is large.

Binary Encoding

Converts categories into binary values, using fewer columns and avoiding some ordinal issues. However, it can lose intuitive meaning for certain categories.

Meta Variable

Meta-variables represent key information relevant to credit evaluation, simplifying raw data for better model performance. Example variables include:

  • Income
  • Loan Amount
  • Past Repayment
  • Job Stability

These variables are processed into higher-level data, such as the Loan-to-Income Ratio, to improve efficiency and accuracy.

Multiple Models

Regression Models

Used for binary classification, predicting probabilities (e.g., 0 or 1).

Random Forest Model

Builds decision trees using random subsets, averaging results for better predictions.

CatBoost

Handles categorical and numerical features efficiently, reducing overfitting and manual preprocessing.

Model Performance Measures

  • False Positive Rate: Measures incorrect positive classifications.
  • Accuracy: Evaluates overall correctness.
  • Elapsed Time: Assesses processing speed.
  • ROC Curve: Plots True Positive Rate against False Positive Rate to assess model ability.

Advantages of AI Credit Scoring

  • Enhancing Financial Inclusion: Enables access for people without credit histories.
  • Comprehensive Risk Assessment: Integrates diverse data like social media and consumer behavior.
  • Improving Loan Efficiency: Automates loan approvals with reliable outputs.

Future Outlook

  • Advancing technology to improve credit scoring.
  • Diversifying data sources for better predictions.
  • Balancing privacy concerns with data analysis.
